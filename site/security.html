Common Vulnerabilities to Review

  • Reentrancy: enforce checks-effects-interactions and reentrancy guards.
  • Access control flaws: verify owner/admin modifiers, role transfer logic, and emergency permissions.
  • Integer precision & overflow issues: validate arithmetic assumptions, token decimals, and rounding edges.
  • Oracle manipulation: use resilient oracle designs, TWAPs, and bounded price validation.
  • Unchecked external calls: handle return values and failure branches explicitly.
  • Denial-of-service vectors: avoid unbounded loops and gas griefing patterns.
  • Signature replay/domain issues: include nonces, chain IDs, and structured signing schemes.

Secure Development Lifecycle

  • Threat modeling at design time (actors, trust boundaries, attack paths).
  • Peer review and static analysis during implementation.
  • Unit/integration/fuzz/property testing before release.
  • Independent security audits before high-value deployment.
  • Post-launch monitoring and incident response playbooks.
  • Responsible disclosure + bug bounty process for continuous hardening.

Checklist baseline: map controls to OWASP Smart Contract Security Verification Standard (SCSVS) and Smart Contract Security Top 10 style checks.

Operator Checklist (Quick Pass)

  • [ ] Privileged functions are role-gated and timelocked where possible.
  • [ ] External dependencies (oracles/bridges) have fallback logic.
  • [ ] Upgradeability controls are documented and tested.
  • [ ] Critical invariants have automated tests and alerting.
  • [ ] Audit findings are resolved or risk-accepted in writing.
